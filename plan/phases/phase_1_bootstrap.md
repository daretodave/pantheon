# Phase 1 — Bootstrap

> **First conscious ship.** This is the only phase the user
> typically runs `/ship-a-phase` for manually before letting
> `/march` take over. It establishes the scaffold every later
> phase relies on. The deploy gate's first green run lands
> here.

## Goal

Stand up a working Next.js 15 app at `pantheon-coral.vercel.app`
that:
- Renders a placeholder home page reading "Pantheon — the
  seasons, ranked. no spoilers." with the ceremonial gold
  primary, paper-tinted dark background.
- Has a working verify gate (`pnpm verify`) — typecheck +
  vitest + build + playwright e2e against `next start :4173`.
- Has a working deploy gate (`pnpm deploy:check`) — green on
  Vercel after first push.
- Carries the Vercel Analytics component, gated on
  `DISABLE_ANALYTICS=1` so e2e doesn't pollute the dataset.
- Generates the favicon set from a placeholder
  `public/sigil.svg` via `scripts/build-icons.mjs`.
- Exposes design tokens from `design/tokens.json` to Tailwind
  via CSS custom properties.

The bar: a competent agent could ship this without asking the
user a question. Every ambiguity below is resolved with an
explicit decision.

### 2. tsconfig + biome + tailwind + next config

Standard Next.js 15 strict-TypeScript scaffold. `tsconfig.json`
has `strict: true`, `noUncheckedIndexedAccess: true`, path
aliases `@/*` → `src/*`. `biome.json` enables the recommended

so the per-show palette swap (lands phase 4) just works.

### 3. `scripts/tokens-to-css.mjs`

Reads `design/tokens.json` and writes `src/styles/tokens.css`.

**FOUC discipline — non-negotiable.** Pantheon's default mode
is dark. The token sheet ships dark tokens at `:root` so SSR
output + first paint = dark, *with no flash*, regardless of
whether JavaScript loads. Light mode only takes effect when
`<html data-theme="light">` is explicitly set by the inline
head script (Section 5c).

Output structure:

```css
/* :root = the default. Dark. SSR sees this. No flash. */
:root {
  --color-paper-0: #0E0B08;
  --color-paper-1: #15110C;
  --color-ink-0:   #F2EADB;
  --color-primary-base: #E8B65A;
  /* ...all dark-mode token bindings... */
}

/* Light mode applies only when explicitly opted in. */
html[data-theme='light'] {
  --color-paper-0: #F5EFE3;
  --color-paper-1: #EEE7D7;
  --color-ink-0:   #1A1610;
  --color-primary-base: #8B5A1F;
  /* ...all light-mode token bindings... */
}

/* Mode-agnostic tokens (spacing, radii, shadows, motion) */
:root {
  --space-1: 4px;
  --space-2: 8px;
  /* ... */
}
```

**Critical:** do NOT put light-mode at `:root` + dark-mode at
`html[data-theme='dark']`. That ordering causes white-flash on
dark-default users (SSR output is light, JS flips to dark on
mount, eye registers a flash). Pantheon flips it: dark at
`:root` is the floor; light is the opt-in override.

Each token surfaces as a CSS custom property:
`--color-paper-0`, `--color-ink-1`, `--color-primary-base`,
`--space-4`, etc.

Run on every `pnpm build` so token changes auto-flow.

### 5. `scripts/build-icons.mjs`

Reads `public/sigil.svg`, emits favicon variants. Uses
`@resvg/resvg-js` (pure-JS) and `png-to-ico`:

```js
import { Resvg } from '@resvg/resvg-js'
import pngToIco from 'png-to-ico'
import { readFile, writeFile } from 'node:fs/promises'

const SVG = await readFile('public/sigil.svg')
const SIZES = [16, 32, 48, 64, 96, 128, 180, 192, 256, 512, 1024]

for (const size of SIZES) {
  const png = new Resvg(SVG, { fitTo: { mode: 'width', value: size } }).render().asPng()
  await writeFile(`public/icon-${size}.png`, png)
}

await writeFile('public/apple-touch-icon.png', /* 180px */ ...)
const ico = await pngToIco([16, 32, 48].map(s => `public/icon-${s}.png`))
await writeFile('public/favicon.ico', ico)
```

Add `@resvg/resvg-js` and `png-to-ico` to devDependencies.

### 5c. ThemeToggle — dark/light switch (no flicker, ever)

`src/components/chrome/ThemeToggle.tsx` — `'use client'`. A
small icon button (sun/moon glyph) that flips
`document.documentElement.dataset.theme` between `'dark'` and
`'light'`. On mount, reads `localStorage.pantheon_theme`;
falls back to `'dark'` if absent. Writes the new value on
every toggle.

**No-flicker discipline — this is the part the user has been
burned on before. Get it right.** Three rules combine to
produce zero FOUC in EITHER direction:

1. **Tokens default dark at `:root`** (Section 3). SSR
   markup + first paint render dark by default with no
   client-side intervention. A user who never enabled
   light mode sees dark on every refresh, no flash.
2. **Light-mode opt-in is set BEFORE first paint** via an
   inline `<script>` in `<head>`, *before* the stylesheet
   link. The script runs synchronously — no `defer`, no
   `async`, no `type="module"` — so the browser sets the
   `data-theme="light"` attribute on `<html>` before it
   evaluates any CSS. Result: light-mode users also see
   their preferred theme on first paint, no flash.
3. **No client-side flipping after mount.** The
   `<ThemeToggle />` component does NOT read localStorage on
   mount and write to dataset.theme. That would cause a
   1-frame mismatch between SSR output and post-mount
   correction. Instead, the inline head script is the
   single source of truth for mount-time theme; the
   component only reflects the current `data-theme`
   attribute and flips it on click.

The inline head script (lives in `src/app/layout.tsx`,
inside `<head>`, BEFORE any stylesheet `<link>` and BEFORE
the React tree):

```tsx
<head>
  {/* No-flicker theme bootstrap. Runs before first paint.
      Must stay synchronous (no async/defer/module). */}
  <script
    dangerouslySetInnerHTML={{
      __html: `
        (function() {
          try {
            var t = localStorage.getItem('pantheon_theme');
            if (t === 'light' || t === 'dark') {
              document.documentElement.dataset.theme = t;
            }
            // No else — :root tokens default dark, no work to do.
          } catch (e) {
            // localStorage blocked (private mode, etc.) — fall
            // through to :root defaults. Still no flash.
          }
        })();
      `,
    }}
  />
  {/* stylesheets, fonts, etc. follow */}
</head>
```

Note: the `<html lang="en" suppressHydrationWarning>`
attribute on the root element is required (the inline script
mutates the dataset, which React would otherwise flag as a
hydration mismatch).

Unit test in `__tests__/ThemeToggle.test.tsx` covers:
- Initial render reads localStorage.
- Click toggles dataset.theme + writes localStorage.
- Defaults to `dark` when localStorage is empty.
- `prefers-reduced-motion` respected on the icon flip.

Accessibility: the toggle button has `aria-label="Switch to
{light|dark} mode"` reflecting the current state's opposite
target. WCAG AA touch target (≥44px).

## Decisions made upfront — DO NOT ASK

- **Package manager:** pnpm. Lock to `9.x`. (Bearings.)
- **Node version:** 22.x (Vercel default for Next.js 15).
- **Single-package layout:** `src/` at root + `apps/e2e/` for
  the Playwright workspace. NOT a full monorepo (no
  pnpm-workspace.yaml). Keep simple until justified.
- **Tailwind:** version 4. Use the new CSS-first config.
- **Theme default:** dark mode default. Light toggle ships
  later (audit row will surface it).
- **Default font:** system stack initially; Source Serif 4 +
  Inter + JetBrains Mono load via `next/font/google` (only
  the weights/subsets actually used).
- **Sentry / error tracking:** none in v1.
- **i18n:** none in v1.
- **PWA / service worker:** none in v1.
- **Image optimization:** Next.js default `<Image>` for any
  raster images that show up later (none in phase 1).
- **The placeholder sigil's design:** stylized "P" — minimal,
  intentionally provisional. Phase 4 ships the real one.
- **`pnpm-lock.yaml` is committed.** Required for hermetic CI.
- **No README content beyond a 3-line placeholder.** README
  fills in over time as `/iterate` finds gaps.

## Failure modes — when to stop

1. `pnpm install` fails 3+ times on the same dependency
   resolution issue → stop, file an issue, do not retry.
2. The first Vercel deploy returns build error → check Vercel
   logs, patch (e.g., missing env var, Node version mismatch),
   re-push. Three iterations max.
3. Playwright can't bind `:4173` → another process is using
   it; kill or change port. Stop if neither works.
4. `tokens-to-css.mjs` can't parse `design/tokens.json` →
   fix the script (the tokens file is canonical).
5. Anything else: stop and report. Phase 1 is too foundational
   to muddle through.
